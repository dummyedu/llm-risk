# 小本体路径

## 是什么

小本体路径是一种进入复杂企业 AI 问题的方法：不要一开始建完整企业世界，而是围绕一个
高价值场景，先把关键对象、关键逻辑和受控行动跑通。

## 为什么重要

完整 ontology 成本高、协作难、验证周期长。小本体可以降低起步复杂度，让团队尽早验证：

- 数据是否足够；
- 逻辑是否抓住关键判断；
- 行动是否可治理；
- 结果是否产生价值。

## 基本步骤

```text
选择高价值场景
-> 定义关键业务对象
-> 接入必要数据
-> 建立关键关系和状态
-> 实现关键 logic
-> 设计受治理 action
-> 验证价值
-> 再扩展对象和场景
```

## 不是目标

小本体不是最终目标。最终目标是业务价值和可持续演化能力。

# 以 Ontology 为中心的交付

## 是什么

以 ontology 为中心的交付，是把业务目标、数据接入、对象建模、逻辑实现、行动治理和用户工作流
放在同一个闭环里推进。

它不是先建立完整企业模型，也不是围绕单次需求写一次性脚本。更合适的路径是：

```text
围绕一个明确客户问题
-> 建立足够支撑判断、action、workflow、权限和审计的局部模型
-> 让这个局部模型在现场跑起来
-> 通过真实反馈继续扩展
```

## 当前假设

交付不应从“大平台建设”开始，而应从一个能验证业务价值的小闭环开始。

FDE 的角色可以暂时理解为平台能力与客户现实之间的前线翻译和实施角色。他不应该试图建立所有模型，
而是围绕客户当前问题建出一个足够可运行、可治理、可复用的局部业务模型。

## FDE 在其中做什么

FDE 补的不是 ontology 定义，而是 ontology 如何进入客户现场。

客户通常不会直接提出“我要建一个 ontology”。客户更常提出的是：

```text
订单处理太慢
信用冻结误伤客户
供应链风险看不清
审批流程反复靠人催
```

FDE 的工作是把这种模糊问题翻译成平台里的运行结构：

```text
业务问题
-> 关键对象
-> 关键属性
-> 对象关系
-> 状态变化
-> action
-> workflow
-> 权限、审批、审计
-> 业务界面和反馈
```

这解释了为什么建模能力是核心能力。工具菜单本身不会告诉团队要做什么功能；功能应从业务模型里长出来。

## 建模边界

模型可以无限展开，所以交付时必须知道在哪里停。

一个实用边界是：

```text
只建模到足以支撑当前决策、动作、协作和追责的程度。
```

如果某个信息会影响判断、触发 action、改变状态、涉及权限审批审计，或会被多个流程复用，就应该认真建模。
如果它只是背景、搜索材料或展示字段，可以先停在记录层或解释层。

可以按四层判断建模深度：

```text
记录层：只是存下来，方便查。
解释层：作为人或 AI 的上下文，不直接触发动作。
决策层：参与判断、评分或推荐。
执行层：触发 action，改变业务系统，必须有权限、审批、审计和回滚。
```

风险越高，模型越细；只是展示或辅助理解，模型可以粗。

## 和普通咨询或实施的差异

FDE 容易被理解为高级咨询或实施角色，这个判断有合理性。但它和普通一次性实施的关键差异应该在这里：

```text
普通实施容易交付脚本、报表或流程配置。
ontology-centered delivery 应该把问题沉淀成可复用、可治理、可扩展的业务对象模型。
```

如果没有强平台，FDE 只是普通实施顾问；如果只有平台而没有现场建模和交付能力，复杂平台也很难被客户自助使用。
因此更稳妥的理解是：

```text
Palantir 的客户价值来自强平台 + 前线工程实施 + 客户现场问题翻译的组合。
```

## 现场客户情况

FDE 的现场价值可以从 NHS 叙事里看到一个表征性切片。

在医疗数据场景里，客户问题不只是“把数据接进来”。FDE 面对的现场包括：

```text
高敏感患者数据
不同医院和社区系统的数据模型差异
数据控制者与数据处理者边界
权限、血缘和审计要求
一线医护人员原本跨多个系统工作的低效流程
```

因此，现场交付不只建业务对象，也要处理治理、信任和用户采用：

```text
业务数据整合
-> 数据责任和访问边界
-> 可追溯的数据血缘
-> 一线用户能使用的工作界面
-> 现场培训和反馈
```

这类材料不应被当作项目效果证明，但可以说明 FDE 的主要价值：把复杂平台、敏感数据和客户真实工作流连接起来。

## 平台可交付性

FDE 能把客户问题建成局部模型和工作流，前提是平台本身能在客户环境里运行。

对 Palantir 这类企业平台，交付不只是业务建模，还包括：

```text
跨商业云、私有云、本地机房和隔离网络部署
持续升级和灰度发布
监控、告警和远程支持
机密网络中的前线代理
合规环境建设
边缘设备上的受限部署
```

Baseline / FDIE 这类基础设施角色补的是平台运行层：

```text
客户业务层：FDE 把客户问题变成业务模型和工作流。
平台运行层：Baseline / FDIE 让 Foundry、Gotham、Apollo 在复杂环境中可部署、可升级、可支持。
```

这也提供了一个重要产品化机制：

```text
一线部署痛点
-> 临时脚本
-> 自动化工具
-> 平台核心能力
```

如果现场问题不能沉淀成平台能力，交付就会长期依赖高强度人力服务。Palantir 的可持续性问题之一，
就是 FDE/FDIE 的一线经验能否不断回流为 Apollo、部署、观测、权限和运维平台能力。

# Logic 执行

## 是什么

Logic 执行指业务规则、模型、优化器、仿真和判断函数如何在 ontology 运行时中被调用。

它解决的问题是：业务专家掌握的关键判断，如何从经验和制度变成系统可以执行、测试、
审计和复用的能力。

## 可能的形式

- 规则；
- 函数；
- 模型；
- 优化器；
- 仿真；
- 审批条件；
- 场景判断流程。

## 和语义编译的关系

[[concepts/semantic-compilation]] 可以看成理解 logic execution 的隐喻：
把对象、关系、函数、规则和权限组织成执行计划。
